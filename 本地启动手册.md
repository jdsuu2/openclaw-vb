# OpenClaw 本地启动手册

> 版本：2026.3.13 | 最后更新：2026-03-20

---

## 1. 环境要求

| 依赖 | 最低版本 | 检查命令 |
|------|---------|---------|
| Node.js | >= 22.16.0 | `node -v` |
| pnpm | >= 10.23.0 | `pnpm -v` |

> Docker **不是必须的**，仅在构建 Docker 镜像时需要。

---

## 2. 安装依赖

```bash
cd openclaw
pnpm install
```

---

## 3. 构建项目

### 3.1 构建主程序（TypeScript → dist/）

```bash
pnpm exec tsdown --no-clean
```

或者让启动脚本自动判断是否需要构建（推荐）：

```bash
pnpm start gateway --allow-unconfigured
# run-node.mjs 会自动检测 src 变更并增量构建
```

### 3.2 构建 A2UI 资源包（首次必须）

```bash
pnpm canvas:a2ui:bundle
```

> 如果跳过此步，后续 `pnpm build:docker` 会报错 "Missing A2UI bundle assets"。
> 本地开发运行 Gateway **不需要**此步骤。

### 3.3 构建 Control UI（Web 仪表盘）

```bash
pnpm --filter openclaw-control-ui build
```

> 输出到 `dist/control-ui/`。Gateway 启动时会自动提供内置的 Control UI。

---

## 4. 启动 Gateway

### 4.1 快速启动（开发模式）

```bash
# Windows (PowerShell)
$env:OPENCLAW_SKIP_CHANNELS=1; node openclaw.mjs gateway --port 18789 --allow-unconfigured

# Windows (Git Bash / WSL)
OPENCLAW_SKIP_CHANNELS=1 node openclaw.mjs gateway --port 18789 --allow-unconfigured

# Linux / macOS
OPENCLAW_SKIP_CHANNELS=1 node openclaw.mjs gateway --port 18789 --allow-unconfigured
```

**参数说明：**

| 参数 | 说明 |
|------|------|
| `--port 18789` | 指定监听端口（默认 3000） |
| `--allow-unconfigured` | 跳过初始配置检查，自动生成 auth token |
| `OPENCLAW_SKIP_CHANNELS=1` | 跳过频道连接（Telegram/Discord 等），纯本地开发用 |

### 4.2 使用 pnpm 脚本启动

```bash
pnpm start gateway --port 18789 --allow-unconfigured
```

> `pnpm start` 等价于 `node scripts/run-node.mjs`，会自动检测源码变更并重新构建。

### 4.3 后台启动（适合长时间运行）

```bash
# Git Bash
nohup node openclaw.mjs gateway --port 18789 --allow-unconfigured > gateway.log 2>&1 &
```

## 5. 访问服务

启动成功后，控制台会输出 auth token。各服务地址如下：

| 服务 | 地址 |
|------|------|
| Control UI (仪表盘) | `http://127.0.0.1:18789` |
| Gateway WebSocket | `ws://127.0.0.1:18789` |
| Health 检查 | `http://127.0.0.1:18789/health` |
| Canvas Host | `http://127.0.0.1:18789/__openclaw__/canvas/` |
| Browser Control | `http://127.0.0.1:18791` |

### 5.1 带 Token 访问仪表盘

首次启动时，控制台会输出类似：

```
Auth token: eafd32f8e046884710b9434cc494cf03d06e8d583f61aacf
```

在浏览器中打开：

```
http://127.0.0.1:18789/#token=<你的token>
```

### 5.2 验证 Gateway 健康状态

```bash
curl http://127.0.0.1:18789/health
# 预期返回：{"ok":true,"status":"live"}
```

## 6. 配置模型提供商

打开仪表盘后，在左侧导航栏点击 **模型配置**，可以：

1. 从预置列表添加 Provider（阿里云百炼、DeepSeek、OpenRouter、Together AI、OpenAI）
2. 或选择"自定义"添加任意 OpenAI 兼容的 API
3. 填写 Base URL 和 API Key
4. 添加自定义模型 ID
5. 选择 Agent 默认模型
6. 点击"保存并重启 Gateway"

## 7. 常用命令速查

```bash
# 安装依赖
pnpm install

# 构建主程序
pnpm exec tsdown --no-clean

# 构建 UI
pnpm --filter openclaw-control-ui build

# 启动 Gateway（开发模式）
OPENCLAW_SKIP_CHANNELS=1 node openclaw.mjs gateway --port 18789 --allow-unconfigured

# 交互式配置向导（配置模型和频道）
node openclaw.mjs onboard

# 查看帮助
node openclaw.mjs --help
node openclaw.mjs gateway --help
```

## 8. 常见问题

### Q: 启动报 "Missing config"
**A:** 添加 `--allow-unconfigured` 参数，或先运行 `node openclaw.mjs setup`。

### Q: 构建报 "Missing A2UI bundle assets"
**A:** 先执行 `pnpm canvas:a2ui:bundle`，再重新构建。

### Q: fetch() 在 bash 子进程中报错
**A:** 这是 Node.js 在某些 shell 环境下的限制，使用 `require('http')` 替代。

### Q: 端口被占用
**A:** 更换端口号：`--port 18790`，或终止占用端口的进程。

### Q: 如何停止 Gateway
**A:** 在终端按 `Ctrl+C`；如果是后台启动的，使用 `kill <PID>`。
